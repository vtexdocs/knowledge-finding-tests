# EDU-17747 — Phase 1: What we learned building the External Search runners

---

## 1. What it was like building and running these tools

We tried **two different approaches** to search Google:

**Approach 1: Google Custom Search API**
- This is an official tool from Google that gives you search results through a web service.
- **Pros:** Easy to understand, predictable results, no robots getting blocked, no surprise page changes.
- **Cons:** Costs money after 100 free searches per day. Setup is tricky (need to enable billing, get API keys, make sure permissions are right). We spent a lot of time debugging "403" errors (permission denied) before realizing the project wasn't properly linked to billing.

**Approach 2: Playwright (Browser Simulation)**
- This opens a real web browser (like you would) and goes to google.com to do the search.
- **Pros:** You see exactly what a human would see, including special features like "AI Overview" (Google's AI-written summary).
- **Cons:** Google blocks automation with CAPTCHA challenges. We had to add a pause button so you can solve the CAPTCHA, then press Enter for the script to continue.

**What we found:**
- API is simple once setup is correct, but Playwright feels more "real" because it shows what users actually see.
- Running many searches takes a long time and needs attention (for CAPTCHAs).
- Focused searches (like "just Portuguese queries" or "just the first 2 issues") are fast and predictable.

---

## 2. Interesting things we noticed in the results

**API results:**
- Very reliable. Always gives 7 links per search with title, snippet, and URL.
- No surprises, no DOM (Document Object Model, aka "webpage structure") changes.
- Good for a stable baseline to compare against.

**Playwright results (and what we had to fix):**

*Problem 1: Too few results*
- Sometimes we got only 1 result when the page should have had 7.
- **Root cause:** The script found one result (like a featured snippet—a highlighted answer box), and stopped looking instead of continuing to find more.
- **Fix:** Keep looking through all available sources until we have 7 results.

*Problem 2: Wrong results*
- Sometimes we were counting Google's own search links as results (like `google.com/search?q=...`).
- **Root cause:** We didn't filter out Google's internal links, only external websites.
- **Fix:** Ignore any link that goes to Google (unless it's webcache).

*Problem 3: Empty descriptions*
- Some results had no title or summary text.
- **Root cause:** We were looking for snippet text in specific HTML locations that sometimes don't exist, and we had no backup plan.
- **Fix:** If there's no title/snippet, create a simple label from the website URL (like `help.vtex.com/en/docs`).

*Bonus: AI Overview*
- We capture the "AI Overview" (Google's AI-written answer) when it appears and save it in our results.
- It's not perfect—Google's page layout changes, so sometimes we miss it. But it's useful when we get it.

## 3. What surprised us (good and bad)

**Good surprises:**
- The API is very cheap and reliable once set up.
- Playwright works well for long searches (over 100 queries) even with CAPTCHAs, if you use the right settings (slower delays, browser profile that remembers cookies).

**Bad surprises:**
- One project being set up in Google Cloud but NOT linked to billing = forever getting permission errors. This wasted a lot of time.
- On Windows, the browser profile sometimes crashes right after starting. We added a fallback: use a temporary browser profile instead (you'll just see more CAPTCHAs, but it works).
- Google changes its webpage structure sometimes. Our selectors (ways to find things on the page) might break if Google redesigns.

---

## 4. Frustrations and limits we hit

**Frustrations:**
1. Google Cloud setup is confusing (need billing, API key, Programmable Search Engine ID, correct project linking).
2. CAPTCHA appears during Playwright runs and you have to solve it and press a button.
3. Browser profile crashes on Windows in some situations.
4. You can't predict when Google will change the page structure.

**Surprises we didn't expect:**
1. Getting only 1 result because we returned too early.
2. Counting Google's own search links as legitimate results.

**Limits we discovered:**
1. Playwright can't guarantee "exactly what a user sees" if CAPTCHA or consent dialogs block the page.
2. AI Overview extraction is "best effort"—it works most of the time but depends on the webpage staying the same.
3. API search results are NOT the same as google.com search results (different ranking algorithm, different index). So comparing them is like comparing apples to oranges.

---

## 5. Ideas for future work

- **Compare API vs Playwright:** Run the same searches on both and see which results overlap. This tells us how different they are.
- **Track AI Overview:** How often does Google show an AI Overview? Does it show up more for certain types of queries?
