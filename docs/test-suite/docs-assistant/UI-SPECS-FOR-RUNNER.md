# Docs-assistant UI specs used by the runner (knowledge-finding-tests)

This document summarizes the specifications from the [docs-assistant](https://github.com/.../docs-assistant) project that the `run-docs-assistant-path.js` script mirrors to evaluate the user experience.

## 1. Source of the final response

- **Origin:** SSE event `ToolOutput` when the tool is `final_answer`; the payload is `data.output` (string).
- **Backend (tools.py):** returns `answer + "\n\n" + "## References\n" + "- [title](url)\n" ... + "\n*Language: {locale} | Confidence: {confidence}*"`.

## 2. Parsing the final response (aligned with `ui/src/lib/answerParser.ts`)

What the UI treats as “what the user sees”:

| Field        | Rule |
|-------------|--------|
| **content** | Everything before `## References`, trimmed. **No character limit.** |
| **references** | Extracted from `## References\n(- [title](url)\n)+` → array `{ title, url }`. |
| **locale**  | Footer `*Language: (\w+) \| Confidence: (\w+)*` → `en` \| `pt` \| `es`. |
| **confidence** | Same footer → `high` \| `medium` \| `low`. |

## 3. What the UI displays

- **Answer block:** only the `content` (answer without the References section and without the footer). Rendered as full markdown, **no truncation**.
- **References block:** below the answer, list of `{ title, url }` (e.g. chips with title + link). Title may be visually truncated (e.g. `max-w-[180px]`) for layout only.
- **locale / confidence:** shown optionally (e.g. in the References component); the UI also uses `locale` to switch the interface language.

## 4. Character limits in the UI

- **Final response (final_answer):** **no** limit; the UI shows the full content.
- **1200 characters:** used only for the **excerpt of the fetch_document tool output** in the collapsible card (`ToolEventCard.tsx`), not for the final response.

## 5. Data format to store (per query)

To mirror the UI in the runner and in the report:

- **answer_content** (string): response text as the UI shows it (equivalent to the parser’s `content`).
- **references**: array of `{ title, url }`.
- **locale** (optional): `en` \| `pt` \| `es`.
- **confidence** (optional): `high` \| `medium` \| `low`.

The runner may also store the raw string `final_answer` (full tool output) for debugging.

## 6. Markdown report structure (analysis-[run_id].md)

The report generated by the run-docs-assistant-path command reflects this spec. **Full report structure** (sections, headings, per-query block with Expected doc URL, Links, Output) is defined in **docs/test-suite/docs-assistant/run-docs-assistant-path.md** (§6 Generate analysis report).

Summary: header and metrics (Response Overview, By Query Style, Errors, Results by Issue); then **Results** — one block per query with Expected doc URL | Found | Rank, **#### Links found by the API**, **#### Output** (answer_content, References, Language | Confidence). Footer with generator and timestamp.
