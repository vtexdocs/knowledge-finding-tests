Comparison vs timeline: comparison is for one change (two runs, deltas). Timeline is for many checkpoints over time (line charts, first→last movers). Phase 1 metrics are target-doc pass rate and MRR on links — not AI answer quality.

How to add a new search audit to the dashboard

  1. Collect search results — Run Phase 1 path runners (internal search, Docs Assistant, external search, LLMs) so raw results exist under results/. The full pipeline can do this for you.
    py tools/test-suite/run_all_runners.py
  2. Process the audit (analysis run) — Joins issue ground truth with collected links and writes aggregates under results/analysis-system/. Use --render-dashboard to build the HTML in the same step.
    py tools/test-suite/analysis_system.py run --render-dashboard
  3. Refresh the runs index — Updates the table on this hub page and cross-links to comparisons.
    py tools/test-suite/render_analysis_dashboard.py --build-index
  4. Compare before vs after (optional) — After a second analysis run, measure deltas between baseline and candidate.
    py tools/test-suite/analysis_system.py compare --baseline "..." --candidate "..." --render-dashboard
  5. Timeline across runs (optional) — Visualize trends when you have several analysis runs (oldest→newest).
    py tools/test-suite/render_analysis_dashboard.py --timeline-runs "run-a" "run-b" "run-c"

Docs: docs/test-suite/analysis-system.md and docs/test-suite/analysis-dashboard.md. Re-render existing HTML after code changes: --rebuild-all-analysis-dashboards or --rebuild-all-comparison-dashboards.

How to read this dashboard
  • Target pass rate — Share of tested queries where the expected target document appeared in the ranked links. Primary success metric for Phase 1 (exact target-doc match).
  • Target MRR — Mean reciprocal rank of the best target doc in the result list. Higher is better (1.0 = always rank 1).
  • Helpful pass rate — Share of tested queries with at least one helpful (relevant, non-unrelated) link in the results.
  • Not available — Queries excluded from pass-rate denominators because the path/locale combination does not apply (e.g. path not run for that locale).
  • Locale gap vs EN — How much ES or PT pass rate differs from EN on each path — highlights locale underperformance.
  • Path families — Discovery paths grouped into Internal search, Docs Assistant, External search, and External LLMs.
  • MRR vs pass rate charts — Chart Y-axis uses target MRR (0–1 scale).

Tip: hover or tap the ? next to labels for definitions.
