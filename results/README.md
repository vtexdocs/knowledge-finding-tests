# Test run results

This directory stores **outputs** from the KR1 Phase 1 path runners: raw JSON, markdown reports, and provider-specific artifacts. Authored documentation lives in `docs/`.

## Canonical layout

All runners now persist files under:

```text
results/<path-family>/<variant>/<run-folder>/...
```

Examples:

- `results/internal-search/algolia-helpcenter/algolia-helpcenter 2026-03-17 14-49/`
- `results/internal-search/hybrid-search/hybrid-search 2026-03-17 14-53/`
- `results/docs-assistant/api/docs-assistant 2026-03-17 16-23/`
- `results/external-search/google-search-playwright/google-search-playwright 2026-03-24 15-37/`
- `results/external-llms/chatgpt/external-llms-chatgpt 2026-03-04 22-26/`
- `results/analysis-system/analysis-system 2026-03-25 13-33 (1)/`
- `results/analysis-system-comparisons/analysis-system 2026-03-25 13-34/`

Variant folders currently used by the repository:

| Path family | Variants |
|--------|----------|
| `internal-search/` | `algolia-helpcenter/`, `algolia-devportal/`, `hybrid-search/` |
| `docs-assistant/` | `api/` |
| `external-search/` | `google-search-playwright/` |
| `external-llms/` | `chatgpt/`, `gemini/` |
| `analysis-system/` | one processed analysis run folder per execution |
| `analysis-system-comparisons/` | one comparison run folder per execution |

## Files inside a run folder

The exact artifact set depends on the runner:

- Internal search: `<run-id>.json`, optional `analysis-<run-id>.md`, and for Hybrid Search optionally `internal-search-comparison-*.md`
- Docs Assistant: `<run-id>.json` and optional `analysis-<run-id>.md`
- External search: `results.json`
- External LLMs: `run_metadata.json` plus one markdown file per `(issue, locale, style)`
- Analysis system: processed JSON artifacts such as `issues_processed.json`, `aggregates_by_path_locale.json`, `failure_list.json`, and `run_summary.json`, plus an optional `dashboard/` folder generated by the current WIP visualization flow
- Analysis system comparisons: comparison JSON artifacts such as `comparison_summary.json` and `comparison_by_path_locale.json`, plus an optional `dashboard/` folder generated by the current WIP visualization flow

## Historical normalization

Older runs used a mix of flat folders and legacy nested trees. The repository includes a migration helper to move those historical artifacts into the canonical layout:

```powershell
python "tools/test-suite/standardize_results_layout.py" --write
```

Dry-run mode is available by omitting `--write`.

## Unified output format

Path runners aim to keep the raw payloads aligned with a unified output format. See [docs/test-suite/results-layout.md](../docs/test-suite/results-layout.md) for the structure used in this repository.
